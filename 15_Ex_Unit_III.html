1 Degree Correlation


  1. C-elegans. Consider the neural network of the C-elegans used in the midterm. In the following link you can find the names of the neurons. Then:

    • Download the names of the neurons and create an attribute of the edges with these names
    • In terms of the degree correlation, is the network assortative or disassortative? If it is disassortative, find out if it is structural or not. For both questions, do the graphical analysis.
    • Find the communities with the fast_greedy, walktrap and edge_betweenness
    • Considering the one with the highest modularity, try to interpret the communities in the network (note that for this you will have to look for the meaning of the names of the neurons here for example). What we are interested in knowing is:
      • Are the communities homogeneous in any sense?
      • Are the main nodes in the different communities relevant in any sense? For this second part, it may be interesting if you filter by communities and find if there are central nodes


  2. Network Science. Use the Network Science Collaborations network and then:

    • Determine if, with respect to the degree, the network is assortative or disassortative. What does it mean?
    • Identify the communities (any algorithm of your choice) and the relevant nodes in each of them (use degree). With the relevant nodes identified, try to look for information of the scientists and give an in-context interpretation


  3. Spectral Decomposition. Generate three models: ER-model, WS-model and BA-model, apart from using 50 nodes, choose the properties you want and then explain the regimes in which each network are. Then:

    • Perform the spectral decomposition
    • Using the K-means clustering method, determine the optimum number of components (communities) that may best describe the structure of the networks. With this information, find the communities and make a plot of each network.


  4. Karate Club. Use the karate dataset and then:

    • Find the community structure by, at least, three different algorithms
    • Select the clustering with the highest modularity
    • Interpret the results


  5. Simulation. Define a function that let’s you generate a dynamical series of networks from a seeded network. It must satisfy that:

    • Have an igraph object as input
    • Nodes are selected randomly for updating
    • Assume that there is a passage of time (one unit) from update to update (to make things easier, assume that chages in two nodes do not co-occur)
    • The change in the network proceeds only by degree, then you may either add or remove edges at each step (if you want to make this using any particular mechanism, do it. If not, a random generation is perfect)
    • The output is an igraph object

    Once you have the starting network and an ending network find:

    • If the networks are assortative, disassortative or neutral (with respect to degree)
    • In any case, perform a Randomization test to determine if the (even if it is small) change in assortativity is due to homophily or to influence (this may depend on the selection mechanism you use in the function above)

2 Case: Twitter

Consider the Twitter Networks for USA Elections. The dataset contains the same Twitter search in two different times, and each contains different nodes and types of links. For the analysis, through filtering, you can choose any type of node (user, tweet,…) and any type of link (retweets from, has mention,…) as long as they produce a network with links. Th easiest options are User and then quotes_from or retweets_from, since the other options will imply working with bimodal/affiliation networks. Then:

  • Find the communities through their modularity class
  • Make the graph using modularity as color (If you have too many isolated nodes you can remove them with the filter topology -> k-cores or Degree Range). Then answer the following:
  • Do you idenfity relevant nodes per community?
  • With respect to which centrality measure do you find them?
  • What does it mean?

Due to the evolution of the network you can find changes in assortativity, the attributes you have may fit good for degree or unordered assortativity, then degree correlation or modularity. Let’s consider the two networks as the same network evolved in time

  • Be sure that the filter in both networks are exactly the same and then import your network into R
  • Find the assortativity index in both time stamps: Does it show assortativity or disassortativity? Can you explain its origin?

As the last part, try to perform a test to identify if the change in the network is due to influence or homophily. Remember that for this you need to generate random networks with the appropriate properties and then perform the test.
